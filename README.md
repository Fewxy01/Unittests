# :sparkles: Написание Unit тестов :sparkles:

## 1. Цели и задачи тестирования

Цель — проверить корректность работы функций вычисления площади и периметра прямоугольника.
Задачи:

- убедиться, что функции area() и perimeter() корректно обрабатывают различные входные данные;

- выявить ошибки в логике вычислений

- обеспечить автоматическую проверку корректности функций.

## 2. Описание тестируемого продукта
Тестируется python-модуль rectangle.py, содержащий формулы вычисления периметра и площади прямоугольника

**rectangle.py**

```
def area(a, b):
    return a * b

def perimeter(a, b):
    return 2 * (a + b)
```
Функции принимают на вход два числовых значения и возвращают результат

## 3. Область тестирования

- один из параметров нулевой
- одинаковые значения параметров a и b
- параметры - положительные числа

## 4. Стратегия тестирования

### Стратегия:
- создание набора входных данных
- выполнение функций с указанными параметрами
- сравнение результатов с ожидаемыми числовыми значениями
- автоматически повторяемый прогон тестов

### Тип тестирования:
- функциональное тестирование
- позитивные кейсы(нет отрицательных чисел)

## 5. Критерии приемки

Программа считается успешно прошедшей тестирование если:
- все тесты выполняются без ошибок
- ни один тест не вернул статус FAIL или ERROR
- вычисления верны (совпадают с ожидаемыми тестированием)

## 6. Ожидаемые результаты

- успешное прохождение всех модульных тестов, что подтверждается выводом unittest со статусом OK
- отсутствие ошибок в вычислениях
- отчет тестовой системы о количестве выполненных тестов, статусе тестов(успешен или провален) и сообщение об ошибках при их наличии
